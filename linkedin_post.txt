Hello everyone, I'm happy to share with you my new hobby project!

I built a distributed microservices system that transforms YouTube songs into karaoke tracks.

The core idea: submit a YouTube URL and get back separated vocals/instruments + synchronized lyrics extracted via AI. Real-time progress tracking via WebSockets, full-text search across songs, and an interactive player to tie it all together.

Architecture consists of 5 specialized FastAPI services:
- Orchestrator handles the workflow and job coordination
- Voice separation service runs Demucs/Spleeter for audio demixing
- Lyrics extraction service integrates OpenAI Whisper for speech-to-text
- Song management service handles all the data (PostgreSQL, Elasticsearch)
- Frontend is React with React Router 7 and React Query

Some interesting challenges I worked through:
- Coordinating async jobs across multiple services without blocking users (distributed job queue with RQ + Redis)
- Keeping everything responsive while processing large audio files in the background
- Getting ML models to run efficiently, especially on CPU... 
- Maintaining type safety end-to-end across TypeScript frontend and Python backend (OpenAPI auto-generation)
- Full observability in a distributed system (OpenTelemetry + Prometheus + Grafana)

The whole stack runs in Docker locally, scales on Kubernetes in production, and sits behind a comprehensive observability layer (metrics, traces, logs).

Live at hatal.cc — full details and code on GitHub.

Curious to hear what other people have built with similar stacks or how they've tackled distributed system challenges.
